What is the final mapped url_prefix?

url_prefix is overridden by register_blueprint over the constructor.

bp = Blueprint("app2", __name__, ... , url_prefix="/app2")
app.register_blueprint(bp, url_prefix="/app2-alias")

Where is template looking?

Reference the templates folder relative to the folder where the blueprint is defined.

bp = Blueprint("app2", __name__, ... , template_folder="templates")

Find here: app2 / templates

However, if the same name exists in the root:templates folder, that seems to take precedence.

  • return render_template("index-app2.html")
  • return render_template("about-app2.html")

Therefore, the official recommendation is to specify the folder using the blueprint name.

  • return render_template("app2/hello.html")
